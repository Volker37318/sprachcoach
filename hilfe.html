Schritt-für-Schritt

Lehrkraft: Klasse starten & Code verteilen
  1. Startseite öffnen, Modul wählen (Grammatik-Trainer / Dialog-Partner).
  2. „Code erzeugen“ (Lehransicht) klicken und Code im Kurs teilen – kein QR.
  3. Teilnehmende geben den Code ein, wählen einen Alias-Namen und starten.
  4. Lehransicht per Lehrercode aktivieren: Bewertungen & typische Fehler im Überblick.

Tipp: Aufgaben werden vollautomatisch korrigiert → mehr Zeit für Unterricht.

Teilnehmende: So starten Sie mit dem Code
  1. Seite öffnen, Startcode eingeben (Kurzanleitung).
  2. Alias-Namen wählen (kein Klarnamen-Zwang).
  3. Niveau wählen (A1–B2) – Inhalte entsprechen dem Referenzwortschatz.
  4. Übungen/Dialogsituationen bearbeiten. Fortschritt wird lokal gespeichert.

Für Sprechen: siehe unten Mikrofon freigeben.

Trial/PRO & Lizenzen freischalten
  • 21-Tage-Test: Button „Kostenlos testen“ → lokales Trial-Token (localStorage).
  • PRO (Teilnehmende): Kauf → Claim mit Bestell-ID/E-Mail (/api/claim) → proToken.
  • PRO (Kursträger): Bündelkauf mit −30 %/TN/Monat. Freischaltung via Webhook & Claim.
  • Laufzeit: Gilt für die gebuchte Zeit und endet automatisch. Preise

Probleme lösen

Mikrofon freigeben (Android & iOS) Audio

Wichtig: Beim ersten Start fragt der Browser nach Zugriff auf das Mikrofon. Bitte Zulassen wählen. Wenn bereits blockiert wurde, so vorgehen:

Android – Chrome

  1. Website öffnen (z. B. Demo Teilnehmer), rechts oben ⋮ Einstellungen → Website-Einstellungen → Mikrofon.
  2. Mikrofon aktivieren und unter „Blockiert“ unsere Domain antippen → Zugriff zulassen.
  3. Zurück zur Seite, Neu laden (Strg+R bzw. Reload-Icon) und beim Hinweis Zulassen.
  4. Falls weiterhin blockiert: Android Einstellungen → Apps → Chrome → Berechtigungen → Mikrofon: Zulassen.

Android – Samsung Internet (falls genutzt)

  1. Menü ≡ → Einstellungen → Websites und Downloads → Website-Berechtigungen → Mikrofon → Zulassen.
  2. Domain prüfen: falls unter „Blockiert“, auf Zulassen umstellen, Seite neu laden.

iOS – Safari

  1. iOS Einstellungen → Safari → Mikrofon → Zulassen (oder „Fragen“).
  2. Optional pro Website: in Safari auf unserer Seite aA (Adressleiste) → Website-Einstellungen → Mikrofon → Zulassen.
  3. Seite neu laden und beim Hinweis Zulassen.

iOS – Chrome

  1. iOS Einstellungen → Datenschutz & Sicherheit → Mikrofon → Chrome aktivieren.
  2. Chrome öffnen, unsere Seite laden und den Zugriff Zulassen.

Tipps: Headset testen; prüfen, ob andere Apps (z. B. Recorder/WhatsApp) das Mikro blockieren; ggf. Gerät/Browser wechseln. Bei Schul-Firewalls: Domains für Audio/KI laut Technik freigeben.

Trial startet nicht / Token verloren Zugang
  • Privatmodus? – Trial-Token liegt lokal (sp_trial_started_at).
  • Seite neu laden; Website-Daten nicht automatisch löschen lassen.
  • Bei Bedarf: neuen Trial anfordern (/api/trial) oder PRO claimen (/api/claim).
„Zugriff verweigert“ oder Inhalte gesperrt PRO
  • Trial abgelaufen? – PRO buchen/claimen; beim Kursträger gilt −30 %/TN.
  • Token prüfen: /api/verify → Status „trial / pro / expired“.
  • Bei Sammelkontingenten: Bestellnummer & E-Mail bereithalten (Support hilft).
Lehransicht fehlt / Code wird nicht akzeptiert Lehrkraft
  • Richtige Instanz/Seite? – Lehrercode ist kursbezogen.
  • Code ohne Leerzeichen; Groß-/Kleinschreibung beachten.
  • Bei Browserwechsel Lehrfreischaltung erneut durchführen.
Zahlung & Rechnung Abrechnung
  • Abwicklung über Digistore24 (Karte, SEPA, PayPal, ggf. Rechnung).
  • Kursträger-Buchung: −30 % pro TN/Monat; Weitergabe an TN durch Kursträger.
  • Rechnungsfragen: Bestell-ID/E-Mail an kvhs@pramann.de.

Tipps & Shortcuts

  • Zoom: Strg++ / − (macOS: ⌘).
  • Neuladen: Strg+R (macOS: ⌘ R).
  • Audio schnell prüfen: System-Sprachrekorder öffnen; Pegel checken.
  • Datenschutz: Alias statt Klarname verwenden.
  • Geräte: Desktop, Tablet, Android, iOS – keine Gerätebindung.
  • Fortschritt: lokal gespeichert; Website-Daten nicht automatisch löschen.
